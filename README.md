GENERAL EXAMINATION CONTEXT 



This laboratory examination assesses your ability to design, develop, and deploy a multi-page professional website using advanced JavaScript functionalities, fully compatible with GitHub Pages as a hosting platform. 



Unlike your previous Prelim laboratory activities (personal portfolio websites), this examination requires you to simulate a real-world industry website, following the assigned industry exam set. Each student is assigned ONLY ONE industry and must strictly follow the scope of that industry.  (see attached Industry Set and open the one assigned to you by your professor)



Your output must demonstrate: 

Practical application of HTML5, CSS3, and JavaScript 
Proper client-side scripting (no server-side technologies) 
Professional layout and user experience 
Correct usage of GitHub Pages deployment 
Functional JavaScript features relevant to the industry 
GENERAL REQUIREMENTS (APPLICABLE TO ALL EXAM SETS) 

1. Website Structure 

Your website must consist of a minimum of FIVE (5) distinct webpages, such as: 
Home 
About / Profile 
Services / Programs / Departments 
Announcements / Events / Projects 
Contact / Inquiry / Membership / Volunteer Page 
Pages must be properly linked using a navigation menu. 
2. JavaScript Requirements (MANDATORY) 

Your website must implement ADVANCED JavaScript functionalities, such as (but not limited to): 

Dynamic content rendering using JavaScript 
Interactive forms with client-side validation 
Modal dialogs, sliders, tabs, or accordions 
Filtering or searching of data (e.g., events, services, departments) 
LocalStorage or SessionStorage usage 
Dynamic announcements, notices, or event lists 
Responsive UI behavior controlled by JavaScript 
NOTE: Static websites with minimal or trivial JavaScript will receive significant deductions. 

3. GitHub Pages Hosting 

Your website MUST be hosted on GitHub Pages 
All files must be properly pushed to your repository 
The deployed site must be publicly accessible 
Broken links, missing assets, or non-working JavaScript will be graded accordingly 
4. Submission 

Submit the GitHub repository link in MS Teams 
The repository must contain and should also be organized into folders for manageability: 
HTML files 
CSS files 
JavaScript files
